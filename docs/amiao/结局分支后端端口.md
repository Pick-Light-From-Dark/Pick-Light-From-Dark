### EndingTriggerTester.cs 接口说明

`EndingTriggerTester.cs` 留了三层公开接口，供后续 Demo 直接调用：

---

#### 1. 场景测试接口（直接显示结局面板）

```csharp
// 当场血量测试死亡结局
public void TriggerDeadEnd(int lives)

// 触发结局一（太阳照常升起）
public void TriggerEnding1(string level1ChoiceId = "not_eat")

// 根据第五关条件判定并触发对应结局
public void TriggerEnding5Branch(int l1, int l2, int l3, int l5, bool card2017, bool card2026, RooftopChoice choice)

```

#### 2. 底层通用接口

```csharp
// 直接实例化并显示任意结局预制体
public void ShowEnding(GameObject prefab, int endingId)

```

#### 3. 判定结果读取

```csharp
// 获取上次判定结果的文本（供外部 UI 显示）
public string GetEvaluateResult()

```

---

### 使用示例（外部 Demo 接入）

```csharp
var tester = FindFirstObjectByType<EndingTriggerTester>();

// 血量耗尽 → 触发死亡结局
tester.TriggerDeadEnd(0);

// 第一关选择"不吃" → 触发结局一
tester.TriggerEnding1("not_eat");

// 第五关满条件 → 自动判定为 6005 北极星并触发
tester.TriggerEnding5Branch(2, 2, 2, 2, true, true, EndingTriggerTester.RooftopChoice.WithFriend);

```

---

> **注意：**
> 这些接口都不依赖 `EndingManager` / `EndingContentPanel` 的老路，内部直接走 `Instantiate(prefab)`，可以在没有完整 UI 管线的测试场景里独立工作。
