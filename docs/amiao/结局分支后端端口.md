### EndingTriggerTester.cs 接口说明

`EndingTriggerTester.cs` 留了三层公开接口，供后续 Demo 直接调用：

---

#### 1. 场景测试接口（直接显示结局面板）

```csharp
// 当场血量测试死亡结局
public void TriggerDeadEnd(int lives)

// 触发结局一（太阳照常升起）
public void TriggerEnding1(string level1ChoiceId = "not_eat")

// 根据第五关条件判定并触发对应结局
public void TriggerEnding5Branch(int l1, int l2, int l3, int l5, bool card2017, bool card2026, RooftopChoice choice)

```

#### 2. 底层通用接口

```csharp
// 直接实例化并显示任意结局预制体
public void ShowEnding(GameObject prefab, int endingId)

```

#### 3. 判定结果读取

```csharp
// 获取上次判定结果的文本（供外部 UI 显示）
public string GetEvaluateResult()

```

---

### 使用示例（外部 Demo 接入）

```csharp
var tester = FindFirstObjectByType<EndingTriggerTester>();

// 血量耗尽 → 触发死亡结局
```

# Fungus 调用方式说明

---

## 一、前提：场景中挂载桥接器

在 `endingBranch.unity` 或其他需要调用结局条件的场景中：

1. 创建一个空 **GameObject**，命名为 `EndingBridge`。
2. 挂载 `EndingConditionBridge` 组件。
3. （可选）勾选 **logOperations** 以查看控制台日志。

---

## 二、记录接口 — 卡牌使用时调用

`RecordCard2017()` / `RecordCard2026()`

**作用：** 在玩家使用对应卡牌时，记录该卡牌已被使用。

**Fungus 中使用 Call Method 命令：**

* **Command:** Call Method
* **Target Game Object:** EndingBridge (挂载 EndingConditionBridge 的物体)
* **Method Name:** `RecordCard2017`

**或使用 Invoke Method（无返回值时两者等价）：**

* **Command:** Invoke Method
* **Target Game Object:** EndingBridge
* **Method Name:** `RecordCard2017`

**使用时机：**

* 第二关使用“分享泡面”卡后 → 调用 `RecordCard2017`
* 第五关使用“寻求宋明月帮助”卡后 → 调用 `RecordCard2026`

---

## 三、查询接口 — 条件判断时调用

`HasUsedCard2017()` / `HasUsedCard2026()`

**作用：** 查询单张卡牌是否已被使用，返回 `bool`。

**Fungus 中使用 Invoke Method 命令 + 保存返回值：**

* **Command:** Invoke Method
* **Target Game Object:** EndingBridge
* **Method Name:** `HasUsedCard2017`
* **Save Return Value:** ☑ true
* **Return Value Variable Key:** `HasCard2017`（← 提前创建的 Boolean 变量）

### `CanShowRooftopChoice()` ← 最常用

**作用：** 检查两卡是否全部使用，返回 `bool`。这是控制天台选项显示的关键。

**Fungus 中使用步骤：**

* **Step 1：创建 Boolean Variable**
* Flowchart → Variables → 添加 Boolean 变量，命名为 `CanShowRooftop`


* **Step 2：Invoke Method 查询**
* **Command:** Invoke Method
* **Target Game Object:** EndingBridge
* **Method Name:** `CanShowRooftopChoice`
* **Save Return Value:** ☑ true
* **Return Value Variable Key:** `CanShowRooftop`


* **Step 3：Menu 命令绑定**
* **Command:** Menu
* Option Text: 独自前往
* Target Block: 天台_独自
* **hideThisOption:** `$CanShowRooftop`（← 绑定变量，false 时隐藏此选项）


* **Command:** Menu
* Option Text: 邀请宋明月
* Target Block: 天台_邀请
* **hideThisOption:** `$CanShowRooftop`（← 同上）





**效果：**

* 两卡都用过 → `CanShowRooftop = true` → 两个选项都显示
* 至少一卡没用过 → `CanShowRooftop = false` → 两个选项都隐藏

---

## 四、管理接口 — 测试/重置用

`ResetCardUsage()`

**作用：** 清空所有卡牌使用记录，测试时使用。

**Fungus 调用：**

* **Command:** Call Method
* **Target Game Object:** EndingBridge
* **Method Name:** `ResetCardUsage`

**使用场景：** 重新开始游戏或测试不同分支前，重置条件。

---

## 五、完整流程示例

1. **【第二关剧情】**
* 玩家使用“分享泡面”卡
* → **Call Method:** `RecordCard2017`
* → 持久化到 `CrossLevelSaveSystem`


2. **【第五关剧情】**
* 玩家使用“寻求宋明月帮助”卡
* → **Call Method:** `RecordCard2026`
* → 持久化到 `CrossLevelSaveSystem`


3. **【天台前判定】**
* → **Invoke Method:** `CanShowRooftopChoice`
* → **Save Return Value** → Boolean Variable: `CanShowRooftop`


4. **【天台选项 Menu】**
* → Option A (独自前往): `hideThisOption = $CanShowRooftop`
* → Option B (邀请宋明月): `hideThisOption = $CanShowRooftop`

## 六、后端代码内部逻辑

| 方法 | 持久化方式 | Fallback |
| --- | --- | --- |
| **RecordCard(int)** | `CrossLevelSaveSystem.Instance.RecordCardUsed(cardId)` | 内存 `HashSet<int>` |
| **HasUsedCard(int)** | `CrossLevelSaveSystem.Instance.LoadEndingData().cardsUsed.Contains(cardId)` | 内存 `HashSet<int>.Contains(cardId)` |

> **注意：** 如果 `CrossLevelSaveSystem` 未初始化（比如在独立测试场景），数据仅保存在内存中，场景切换或重启后会丢失。在完整 Demo 中，`CrossLevelSaveSystem` 作为全局单例会在游戏开始时初始化，数据可跨关卡持久化。
