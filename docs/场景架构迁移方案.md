# 场景架构迁移方案

## 一、目标

将当前的单场景 + 代码驱动架构，迁移为**一关一个场景**的架构。GameScene 仅作为主菜单，关卡通过独立场景管理。

## 二、场景规划

```
Scenes/
├── GameScene.unity          ← 主菜单（只加载 BeginPanel）
├── Level1.unity             ← 第一关
├── Level2.unity             ← 第二关
├── Level3.unity             ← 第三关
├── Level4.unity             ← 第四关
└── Level5.unity             ← 第五关
```

## 三、各场景内容

### 3.1 GameScene（主菜单）

**保留现有内容**，不变：
- Camera
- EventSystem
- BeginPanel 通过 UIMgr 动态加载

**改动**：BeginPanel 的 StartBtn 从调用 `LevelFlowManager` 改为场景跳转。

### 3.2 Level 场景（以 Level1 为例）

```
Level1.unity
├── Main Camera
├── EventSystem
├── FungusVN_1 (prefab 实例)
│   └── FungusVNController
│       ├── dialogMask      → dialog_dark.png
│       ├── chineseFont     → 文软雅黑
│       ├── sayDialog       → SayDialog
│       ├── characters[]    → [陆萤, 妈妈, 舍友A, ...]
│       └── backgrounds[]   → [light_room, corridor, ...]
└── LevelFlowCoordinator
    ├── openingStory         → Dialogue1_pre.txt
    ├── endingStory          → Dialogue1_post_eat.txt / Dialogue1_post_noeat.txt
    ├── levelConfig          → LevelConfig_1
    └── vnController         → 场景中的 FungusVN_1
```

每个 Level 场景只配两个东西：
1. **FungusVN prefab** — 提供 Fungus 引擎 + SayDialog + 对话框样式
2. **LevelFlowCoordinator** — 三段式流程编排（开场剧情 → 游玩 → 结尾剧情）

所有 Gameplay UI（GamePanel、TeacherStatus 等）继续通过 UIMgr 从 Resources 动态加载，与场景无关。

## 四、完整运行流程

```
GameScene
  │
  ├─ UIMgr.ShowPanel<BeginPanel>()
  ├─ 玩家点击 [开始游戏]
  └─ SceneManager.LoadScene("Level1")
      │
      Level1 场景加载 ──────────────────────────────
      │
      ├─ LevelFlowCoordinator.Start()
      │   │
      │   ├─ 【阶段1：开场剧情】
      │   │   ├─ FungusVNController.SetupDialogMask()  → 应用 dialog_dark.png
      │   │   ├─ FungusVNController.SetupFont()         → 应用中文字体
      │   │   └─ FungusVNController 播放 openingStory
      │   │       └─ 逐句显示对话，玩家点击推进
      │   │
      │   ├─ 【阶段2：游玩】
      │   │   ├─ UIMgr.ShowPanel<GamePanel>()           → 加载游戏面板
      │   │   ├─ GamePanel.InitializeWithConfig()       → 初始化所有子系统
      │   │   ├─ 创建 TeacherAI GameObject              → 老师巡逻
      │   │   └─ GameFlowController 管理时间/情绪/任务
      │   │       │
      │   │       ├─ 玩家打牌、闭眼、躲老师...
      │   │       ├─ 任务完成 → GameWin
      │   │       └─ 被抓次数耗尽 → GameLose
      │   │
      │   ├─ 【GameWin 路线】
      │   │   ├─ CleanupTeacherAI()
      │   │   ├─ UIMgr.HidePanel<GamePanel>()
      │   │   └─ 【阶段3：结尾剧情】
      │   │       └─ FungusVNController 播放 endingStory
      │   │           └─ 播完 → EndingContentPanel 结算
      │   │               └─ [继续] → SceneManager.LoadScene("Level2")
      │   │
      │   └─ 【GameLose 路线】
      │       ├─ CleanupTeacherAI()
      │       ├─ UIMgr.HidePanel<GamePanel>()
      │       └─ TipPanel 失败提示
      │           └─ [重试] → SceneManager.LoadScene("Level1")
      │           └─ [返回主菜单] → SceneManager.LoadScene("GameScene")
```

## 七、场景切换时的清理

`SceneManager.LoadScene` 会自动销毁旧场景的所有 GameObject（Camera、FungusVN、LevelFlowCoordinator 等）。

需要注意 DontDestroyOnLoad 的对象：
- **UIMgr** — DontDestroyOnLoad，场景切换不销毁，但 `ShowPanel/HidePanel` 不会自动清空已有面板。需要在场景切换前主动 `HideAllPanels()`。
- **FungusManager / SayDialog** — DontDestroyOnLoad，跨场景保留，新场景的 FungusVNController 会重新绑定。
- **GameFlowController / EmotionSystem 等单例** — DontDestroyOnLoad，但 LevelFlowCoordinator 在新场景会重新创建初始化，需确保旧单例被清理。

建议在 `LevelFlowCoordinator.Start()` 开头加清理逻辑：
```csharp
// 清理 DontDestroyOnLoad 中的旧 UI 面板
UIMgr.Instance.HideAllPanels();
```

## 九、风险与注意事项

1. **Fungus SayDialog 跨场景问题**：SayDialog 是 DontDestroyOnLoad，场景切换时不会销毁。LevelFlowCoordinator 的 `Start()` 中 `FindObjectOfType<FungusVNController>()` 可能找到旧的。解决方案：每个场景的 FungusVN prefab 确保 Awake 时重新绑定。

2. **单例残留**：GameFlowController、EmotionSystem 等是 DontDestroyOnLoad 单例，场景切换时不会自动销毁。LevelFlowCoordinator 的 `InitializeWithConfig()` 会重新初始化它们，但要确保旧的监听事件被清理。

3. **UIMgr 面板残留**：场景切换时 UIMgr 管理的面板不会自动销毁。需要在跳转场景前调用清理。
