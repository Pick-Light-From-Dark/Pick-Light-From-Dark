一、游戏流程设计
游戏流程为：进入关卡→剧情流程→游玩流程→剧情流程→前往下一关或前往结局。当开始第一关或进入下一关时，并不会直接进入游玩流程，而是会进入剧情流程，流程切换时需强制清空上一阶段残留 UI 与未执行完的逻辑，禁止跨流程逻辑残留；在剧情流程中，只会显示游戏画面、对话框、CG 图、选择框等 UI，强制隐藏所有核心玩法相关 UI，且全程阻塞游玩逻辑、玩家自由操作，剧情内选择框弹出时需暂停流程推进，仅玩家完成选择后才可继续后续流程；在游玩流程中，会正常显示核心玩法相关 UI，包括 Q 版小人 UI、卡牌 UI、任务栏 UI、倒计时 UI、血量 UI、对话框 UI、退出键 UI、卡槽 UI、情绪值 UI 等，强制隐藏剧情专属 CG、漫画格、剧情选择框 UI，关卡结局分支需明确触发条件，满足对应条件后跳转下一关或结局。
玩家中途退出、重启游戏、重新进入当前关卡，需从当前关卡开头重新开始游玩，清空关卡内所有流程进度、操作记录及临时状态，不保留任何流程断点，回归关卡初始流程节点重新推进。
二、游戏画面系统
1.	画面显示规则：可以在剧情流程和游玩流程显示和隐藏游戏画面，不同流程画面显示互不干扰，显示 / 隐藏操作需同步匹配当前流程状态。
2.	画面切换规则：画面切换通过画面管理类实现，切换过程中阻塞所有新的画面切换请求，禁止打断过渡动画，前一张画面通过一段时间的过渡切换到下一张画面，过渡参数支持配置，过渡完成后再响应后续操作。过渡方式有：
o	硬切：不需要时间等待，直接切换，无过渡时长
o	渐变：前一张图片逐渐隐藏，后一张图片逐渐浮现，通过调节透明度实现，渐变时长可配置
o	动画：前一张画面进行缩放移动同时调节透明度切入下一张图片，实现动态感的效果，缩放位移轨迹、动画时长、透明度变化速率均支持配置
所有画面需明确层级关系，画面层级从高到低固定为：引导遮罩层 > 剧情 CG 层 > 对话框层 > 核心玩法 UI 层（卡牌、任务栏、倒计时等） > 游玩背景层，避免遮挡错乱。
3.	关卡重启画面规则：玩家退出、重启游戏重进关卡时，立即终止所有画面过渡动画，重置所有画面显示状态，加载关卡初始画面，恢复默认层级配置。
三、对话系统
1.	对话系统功能规则：通过在游戏过程中调用对话系统类来显示对话人名字，对话框和说话内容，隐藏时需要将所有内容全部隐藏，切换流程、结束对话、退出关卡、重启游戏重进关卡时需强制清空未显示完的文字、重置对话状态至初始值。
2.	对话显示规则：当触发对话时，对话的文字内容不会一次性全部出现，会逐字出现，文字播放速度支持配置。
3.	对话状态规则：对话状态分为两类，一类为正在对话，一类为完成对话。正在对话则代表对话的文字还没有完整显示，还在显示过程中，此时玩家不可以通过点击屏幕任意位置实现跳过对话或显示完整对话功能；当对话内容完整显示，对话状态则会进入完成对话状态，此时玩家可以点击屏幕任意位置，触发下一个对话，同一时间仅允许一段对话播放，禁止多段对话、多类型对话叠加触发。
4.	对话分类规则：对话分为两类，一类是语言对话，一类是内心独白。当对话为语言对话时，对话框为语言对话框 UI，正常显示说话人名称；当对话为内心独白时，对话框为思考对话框 UI，不显示说话人名称，两类对话可连续衔接，切换时自动重置对话框样式。
5.	说话人规则：说话的人物不止一个，通过一个说话人 ID 管理说话人，当说话的人说话时，对话框中的人物名字显示为对应说话人，ID 匹配错误时默认不显示说话人名称，避免显示异常。
6.	流程联动规则：对话播放期间，阻塞剧情、CG 漫画格、游玩流程推进，全局时间倒计时、老师巡逻倒计时、情绪值自然变化全部暂停，待当前对话完成后再继续后续流程。
7.	关卡重启对话规则：退出关卡、重启游戏重进关卡时，立即终止所有对话播放，清空对话文本、说话人信息，重置对话状态为初始未触发状态，删除临时对话缓存。
四、引导系统
1.	引导系统使用规则：等玩家在此存档中首次推进到此处时，进入游戏引导流程，引导完成后写入存档永久标记，后续不再重复触发，引导优先级高于剧情、游玩流程，引导期间阻塞其他所有流程逻辑。
2.	引导规则：游戏引导开启后进入全局锁定状态，禁用玩家点击操作和键盘快捷键操作，仅保留引导指定交互功能；采用单步线性串行流程，每一步仅开放当前指定唯一目标物体 / 按钮的射线与交互权限，其余所有可交互对象全部禁用，同时全屏覆盖半透明遮罩并镂空当前引导目标区域，搭配高亮、箭头或手指指引标识及引导文案，引导期间屏蔽剧情 UI、CG 漫画格等非引导相关 UI；玩家仅执行当前引导指定操作时判定本步骤完成，自动进入下一引导步骤，误点击空白区域或非目标位置均无任何响应、不跳转不中断流程；所有引导步骤未完成前不可跳步、不可回退、不可手动跳过，玩家长时间无操作时保持当前引导步骤，不自动推进；全部步骤走完后统一解除全局操作锁定，恢复玩家所有操作权限。重启游戏、重新进入关卡时，引导流程不保留断点，需要从头开始游玩关卡并重新进行新手引导流程。引导期间，全局时间倒计时、老师巡逻倒计时、情绪值自然变化全部暂停。
五、音效系统
1.	音效架构逻辑：全局音效采用统一事件监听架构，由全局音效管理器集中管控所有音频播放、暂停、终止逻辑，业务模块仅触发对应音效事件，实现业务与音频逻辑解耦。
2.	音效分类逻辑：音效共分为四大类：卡牌触发音效、对话音效、巡逻音效、UI 音效。
3.	卡牌触发音效逻辑：采用全局事件监听机制实现，每张卡牌配置唯一卡牌 ID，且与专属音效 ID 形成一一对应绑定关系，卡牌数据与音效资源通过 ID 关联配置。
o	播放触发：玩家将卡牌拖入思考框并松手、确认使用卡牌时，卡牌模块触发卡牌使用音效事件，全局音效管理器接收事件后，立即播放对应卡牌 ID 绑定的卡牌触发音效
o	打断处理：卡牌使用过程中被强制打断时，卡牌模块触发卡牌音效终止事件，全局音效管理器接收到事件后，立刻停止该卡牌对应的当前音效
o	正常完成：卡牌读条完成后，全局音效管理器自动停止该卡牌对应的当前音效
o	重复使用：被打断后的卡牌再次被正常使用时，重新触发卡牌使用音效事件，音效从头开始重新播放，无进度接续
4.	对话音效逻辑：基于事件监听模式执行。每次对话框加载文本、开始逐字显示文字时，对话模块触发对话音效播放事件，全局音效管理器接收事件后播放对应对话音效；直至对话框文本内容完全播放完毕，自动触发对话音效终止事件，音效管理器停止当前对话音效。
5.	巡逻音效逻辑：巡逻音效归属于场景环境音效，由全局音效管理器统一调度，分为老师巡逻脚步声、环境杂音两个子类别，全程通过状态事件触发管控。
o	老师巡逻脚步声：脚步声播放规则与老师巡逻移动时长动态绑定：巡逻耗时越短，脚步声播放频率越快，音量起伏节奏越急促；巡逻耗时越长，脚步声播放频率越慢，音量变化节奏越平缓。巡逻开始时触发巡逻脚步声播放事件，巡逻结束时触发脚步声终止事件，音效管理器同步执行对应操作
o	环境杂音：分为正常杂音与无巡逻杂音两类：正常杂音对局全程随机触发播放，由音效管理器按随机频率循环调度；无巡逻杂音仅在老师处于非巡逻状态时触发对应播放事件，该音效一旦启动，需完整播放完毕，不会被后续触发的巡逻脚步声事件强制打断
6.	UI 音效逻辑：UI 音效采用统一交互事件监听机制，分为按钮悬浮、按钮点击两类基础交互音效。仅当 UI 按钮处于可用状态时，鼠标光标悬停触发 UI 按钮悬浮音效事件，鼠标点击触发 UI 按钮点击音效事件；禁用状态的 UI 不响应任何音效事件。所有 UI 音效均由全局音效管理器统一接收事件并执行播放。
六、任务栏系统
1.	关卡绑定规则：进入任意游戏关卡时，程序自动读取当前关卡唯一序号 / 关卡 ID，读取本地配置表中该关卡绑定的专属任务组数据。若读取失败或无对应配置，任务栏默认显示 "暂无任务" 占位文本，避免空白报错。
2.	任务初始排序规则：关卡任务拥有固定配置顺序，进入关卡首次加载时：所有未完成的非隐藏任务按配置顺序从上至下排列，无已完成任务；隐藏任务初始不显示在任务列表中。若为单局内复活 / 原地重试，则按当前已完成状态排序（未完成在上、已完成在下）。
任务状态定义（固定三种）：
o	未完成：正常白色 / 默认色字体，无删除线，排在列表上方
o	已完成：字体降低透明度改为浅色，文字添加删除线，自动归类至任务列表底部
o	隐藏任务：初始不在任务 UI 中显示，后台运行完成条件监听逻辑
3.	任务完成交互 & 排序逻辑：任意单个非隐藏任务达成完成条件后，立即触发以下连锁表现：
① 任务状态变更为「已完成」
② 字体自动切换为浅色 + 删除线样式
③ 该任务即时重新排序，移动至整个任务列表的最末尾
④ 其余未完成任务自动向上补位，保持原有先后顺序不变
⑤ 当所有非隐藏任务均变更为已完成状态时，立即触发游玩流程结束逻辑，强制切换至剧情流程，流程切换严格执行本文档 "一、游戏流程设计" 中的全部规则（清空上一阶段残留 UI 与未执行逻辑、隐藏所有核心玩法相关 UI、阻塞游玩逻辑等）
4.	列表刷新与动画规则：任务状态变更时统一触发一次任务列表整体刷新；任务位置移动、字体样式切换使用瞬时切换，任务数量超出 UI 容器范围时，开启纵向滚动视图。
5.	关卡重置 / 复活 / 重进规则：单局关卡内：玩家死亡、复活、原地重试，仅发生在游玩流程阶段，保留本局所有任务完成状态与排序；一旦触发剧情流程，此关卡将结束游玩流程，不存在复活、重试及回退至游玩流程的可能。主动退出关卡、返回主界面、重新选择进入关卡：所有任务状态（含隐藏任务）全部重置，恢复初始排序与未完成状态，隐藏任务重新进入后台监听模式。
6.	任务监听逻辑（Bug 修复版）：所有任务需在配置表中绑定「卡牌 ID」字段，任务完成条件按卡牌类型区分：
o	不可堆叠类卡牌：当绑定的指定卡牌 ID 成功完成读条并被消耗时，判定为满足任务完成条件
o	可堆叠类卡牌：当绑定的指定卡牌 ID 的剩余层数扣除至 0时，判定为满足任务完成条件
满足条件后触发对应任务状态变更。任务状态一旦变更为「已完成」，立即停止对该任务绑定卡牌的所有事件监听，不再响应任何触发。
7.	隐藏任务规则：隐藏任务全程在后台独立监听完成条件，初始不显示在任务 UI 中；隐藏任务达成完成条件后，自动解除隐藏并显示在任务列表中，位置固定为未完成任务部分之后、已完成任务部分之前；已完成的隐藏任务统一使用紫色背景标识，字体样式与普通已完成任务一致（浅色 + 删除线）；多个隐藏任务同时完成时，按配置表中的固定顺序从上至下排列在中间区域；隐藏任务的完成与否不影响 "所有任务完成触发剧情" 的判定逻辑。
七、全局暂停系统
1.	触发方式：玩家可通过键盘 ESC 键或移动端右上角暂停按钮触发暂停。
2.	暂停效果：暂停期间，所有游戏逻辑（全局时间倒计时、老师巡逻倒计时、情绪值变化、所有音效播放）全部暂停。
3.	暂停菜单：暂停时显示半透明遮罩 + 暂停菜单，提供以下选项：
o	继续游戏
o	重新开始
o	存档
o	设置
o	返回主界面
4.	限制条件：剧情流程、对话播放、新手引导期间无法触发暂停。
八、行动卡
1.	卡牌的移动规则：玩家可以拖动备选区中的卡牌在全屏幕移动，当拖入思考框中并且松手时，卡牌会停留在思考框中，并且开始读条此卡牌。卡牌拖入思考框但未松手时，可拖拽移出思考框，取消本次行动。
2.	卡牌读条规则：当卡牌开始读条时，卡牌上方出现一个时间条，时间条中用绿色显示可打断片段，用红色显示不可打断片段，时间条中的内容从左向右消失，消失时间与卡牌本身的使用时间对应。当卡牌开始读条时，立即隐藏备选区中所有卡牌，此时不可以拖动任何卡牌，且整个读条期间无法触发闭眼操作。读条完成后此卡牌消失，卡牌上方读条 UI 一同消失，增加或减少对应情绪值，立即清空备选区，然后从全局卡牌池中筛选出当前卡牌的所有关联卡牌中「已解锁且剩余层数 > 0」的卡牌，按 ID 从小到大排序显示在备选区中，解锁卡牌操作，玩家可拖动其他卡牌进行下一步操作。
3.	卡牌倒计时片段规则：每张卡牌的倒计时时间不一定相同，总时长片段分为可打断片段和不可打断片段。每张卡牌的两种片段在总时长内的排列顺序、占比数量均不相同。
4.	卡牌打断规则：在可打断片段期间玩家可以点击打断按钮或按键盘空格键，打断正在进行的操作；不可打断片段期间点击打断按钮或按空格键无效。当卡牌被打断时会增加慌乱值，同时将卡牌退回全局卡牌池（可堆叠类层数不变，不可堆叠类保留），清空备选区，恢复读条前的备选区卡牌，不触发任何关联卡牌的解锁或修改，也不会触发顶替机制。
5.	卡牌点击规则：点击卡牌后会显示卡牌的详细信息（卡牌作用、卡牌描述、打断时增加的情绪值等），同时暂停游戏时间，无暂停次数与时长限制。
6.	卡牌显示规则：每局游戏开始时，显示当前关卡对应的初始卡牌，卡牌排列顺序按卡牌 ID 从小到大排序。
7.	被抓时卡牌逻辑规则（已修复）：当玩家被抓时，卡牌读条强制中断，隐藏所有卡牌，一段黑场后，玩家重新回到床上。全局卡牌池所有状态永久保留（包括所有卡牌的已解锁标记、剩余层数、非持续关联顶替关系），仅执行以下操作：
o	清空备选区
o	重新发放本关卡的初始卡牌：遍历初始卡牌列表，对于持续关联类初始卡牌，无论当前层数多少，统一重置为初始层数；对于非持续关联类初始卡牌，仅将剩余层数 > 0 的卡牌显示在备选区
o	所有已解锁的非初始卡牌（通过关联生成的卡牌）暂时隐藏，需通过使用初始卡牌重新触发关联才能再次显示
o	情绪值重置为 50
8.	关联卡牌逻辑规则：卡牌关联类型分为持续关联类、非持续关联类。
o	9.1 持续关联类：玩家使用卡牌后，若该关联卡牌未在全局池，添加到全局池并设置为初始层数；若已在全局池且层数 > 0，保持现有层数不变；若已在全局池且层数 = 0，重置层数为初始层数。可通过其他卡牌的关联效果重复生成 / 重置。
o	9.2 非持续关联类：玩家使用卡牌后，若该关联卡牌从未被解锁过，添加到全局池并设置为初始层数，同时标记为 "已解锁"；若已被解锁过，无论是否在全局池（无论有没有用完），都不再做任何操作。单张卡牌可配置多张不同关联卡牌。
o	9.3 关联触发优化：遍历关联卡牌时，自动跳过已被消耗且已触发顶替的非持续关联卡牌，直接显示其顶替的后续关联卡牌。
o	9.4 非持续关联顶替机制：当一张非持续关联卡牌完成读条并被消耗后，系统会自动将该卡牌的所有关联卡牌，永久添加到该卡牌的所有父卡牌的关联列表中。下次父卡牌触发关联时，会直接显示这些顶替的关联卡牌，而不再显示已被消耗的原非持续关联卡牌。顶替关系为全局永久生效，本局游戏内不会被清除。
9.	卡牌发放规则：游戏开局自动发放当前关卡专属初始卡牌到全局卡牌池；局内新卡牌仅可通过使用卡牌、解锁关联卡牌加入全局池获取。
10.	备选区设计规则：备选区卡牌以卡牌 ID 为顺序从上到下，从右到左显示卡牌，在屏幕中最多显示 10 张，每列 5 张，玩家可以用鼠标上下拖动备选区查看剩余卡牌。
11.	全局卡牌状态池规则
o	游戏全程维护一个唯一的全局卡牌池，记录所有卡牌的解锁状态、剩余层数
o	所有卡牌在全局池中只有一个实例，不存在 "多次生成多个副本" 的情况
o	全局卡牌池状态本局游戏内永久保留，不会因被抓而重置
o	备选区仅显示当前操作流程中可用的卡牌，不直接显示全局所有已解锁卡牌
o	玩家主动退出关卡、重启游戏、重新选择进入关卡时，全局卡牌池完全重置为初始状态，所有已解锁标记、剩余层数、顶替关系全部清除
12.	卡牌堆叠规则：卡牌分为可堆叠类与不可堆叠类。
o	可堆叠类卡牌仅能通过其他卡牌的关联效果生成，初始关卡卡牌中不包含可堆叠类卡牌；不可堆叠类卡牌遵循原有常规卡牌生成逻辑。
o	可堆叠类卡牌生成时，同名卡牌会自动叠放合并，在备选区内仅占用单个卡位，卡牌右上角显示当前堆叠层数；堆叠卡牌均为完全相同的卡牌，点击查看详情时，仅展示单张卡牌的基础信息。
o	所有可堆叠类卡牌统一归类为非持续关联卡牌，不可设置为持续关联类型；堆叠卡牌不会触发任何关联卡牌生成逻辑。
o	玩家拖动备选区中的堆叠卡牌时，默认单独取出单张卡牌进行行动，不消耗整组堆叠。卡牌读条完成后，仅扣除该堆叠卡牌的一层层数；层数扣除至零时，该堆叠卡牌从全局卡牌池移除。
o	堆叠卡牌生效时，情绪值仅按照单张卡牌的数值进行结算。
o	堆叠卡牌不存在堆叠上限限制，可根据卡牌自身的初始层数自由累积。
o	若堆叠卡牌在行动中被打断，取出的单张卡牌将退回堆叠组，堆叠层数自动恢复。
o	每一张堆叠卡牌在使用后，都会独立单独记入历史卡牌记录，遵循原有历史卡牌查看规则。
o	不可堆叠类卡牌每次生成仅保留单张，无法进行叠放，遵循原有全部卡牌基础规则。
九、情绪值
1.	情绪值规则：每局游戏开局加载对应初始情绪数值。情绪值固定区间：总和最小值 30、总和最大值 100，所有增减数值先分别计算慌乱值和兴奋值的变化，再计算总和；若总和 < 30，将总和强制设为 30，慌乱值和兴奋值按当前比例**同步放大；若总和 > 100，将总和强制设为 100，慌乱值和兴奋值按当前比例**同步缩小。慌乱值与兴奋值单个数值下限为 0，可低于 30。
2.	情绪值种类规则：情绪值包含慌乱值、兴奋值两项数值。玩家使用卡牌会触发对应情绪值增减，被老师手电筒照射时会持续增加慌乱值，照射时长与慌乱值增长数值成正比。情绪值实时增减会以浮动数值形式展示在屏幕左下角。兴奋值无独立惩罚效果，仅与慌乱值共同参与临界情绪值判定，用于区分不同卡牌的效果差异。
3.	临界情绪值规则：所有查寝类型（眼神查看、手电筒查看）均生效，当玩家慌乱值与兴奋值之和≥当前关卡临界数值时，直接判定为被抓住。每一关临界情绪值独立配置，关卡难度越高，临界情绪值数值越低。
4.	情绪值图标点击规则：点击界面情绪值图标，可展开查看慌乱值、兴奋值的具体实时数值。
5.	情绪值图标显示规则：当情绪值总和在 30-50 之间时，图标为绿色图标，在 50 - 关卡临界值之间为黄色图标，大于关卡临界值为红色图标。
十、时间
全局时间倒计时规则：每一关拥有独立关卡倒计时，关卡开局即刻开始计时，倒计时结束若未完成关卡任务，直接判定任务失败。
十一、闭眼
闭眼操作：键盘快捷键 C 键，移动端为屏幕左侧中心按钮，点击即可切换睁眼 / 闭眼状态。睁眼闭眼无额外次数与冷却限制。闭眼后屏幕直接黑屏，仅显示中央睁眼按钮，自动隐藏备选区、所有卡牌 UI、打断按钮、任务栏等所有核心玩法 UI；睁眼后立即恢复所有 UI 显示。闭眼期间无法使用卡牌。整个卡牌读条期间（包括可打断片段与不可打断片段）均无法触发闭眼操作。闭眼状态下会持续降低慌乱值，基础速率为 1 点 / 秒，具体数值可按关卡配置调整；若闭眼时长超过阈值，全局游戏时间流速会提升固定倍数，阈值与加速倍数为可配置参数，便于后续数值调优；加速效果持续至玩家切换为睁眼状态。
十二、生命值
单关卡玩家固定拥有 2 点生命值，每次被老师捉住扣除 1 点生命值，生命值归零直接判定游戏失败，触发失败剧情流程。
十三、老师
1.	巡逻状态规则：老师常驻两种状态：未巡逻状态、巡逻中状态。
o	对局开局默认进入未巡逻状态；进入未巡逻状态时，从配置的巡逻时间间隔区间随机取值进行倒计时，倒计时结束自动切换为巡逻中状态。
o	老师进入巡逻中状态时，从巡逻时间值区间随机抽取数值，作为前往寝室门口的移动耗时（接近时长）；抵达门口后执行查寝行为；查寝结束后，间隔与接近时长同等的时长，再次重新触发巡逻循环。
2.	查寝行为规则：查寝分为眼神查看、手电筒查看两种形式，两种查寝行为时长独立配置、数值不固定，各关卡自定义参数。玩家可通过使用「卧床」「离床」卡牌切换卧床 / 离床状态，对局初始状态为卧床。
o	查寝类型在老师开始移动时一次性确定，移动过程中玩家的任何操作都不会改变本次查寝的类型
o	眼神查看触发条件：老师开始移动时，玩家无任何卡牌在读条操作。判定规则：同步检测玩家是否躺在床上、是否闭眼、慌乱值与兴奋值之和是否低于临界值。三者全部满足判定安全；任意一个条件不满足，直接判定被抓住。
o	手电筒查看触发条件：老师开始移动时，玩家存在正在读条的卡牌操作。判定规则：同步检测玩家是否躺在床上、是否闭眼、慌乱值与兴奋值之和是否低于临界值。三者全部满足判定安全；任意一个条件不满足，直接判定被抓住。
o	老师执行查寝行为的全程，禁止玩家拖拽卡牌进入思考框。
3.	巡逻声音规则：完全遵循本规则 "五、音效系统" 中第 5 条 "巡逻音效逻辑" 执行。
4.	老师难度调整规则：提升关卡难度时，同步缩短巡逻时间间隔、缩短巡逻移动耗时、延长手电筒照射持续时长。
十四、难度系统
1.	难度选项规则：游戏提供简单、普通、困难三种难度选项，玩家可在主菜单选择对应难度开始游戏。
2.	难度影响规则：难度影响同一关卡的参数配置，包括但不限于临界情绪值、巡逻时间间隔、巡逻移动耗时、手电筒照射持续时长。难度越高，关卡挑战难度越大。
